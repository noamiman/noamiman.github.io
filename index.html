RouteMaker

RouteMaker is an end-to-end travel planning platform that converts unstructured travel-blog content into structured, scored destination data, then exposes it through an interactive itinerary planner.

Instead of relying on a single rating, users score destinations by travel style (adventure, food, culture, relaxation), build multi-day routes, and export complete plans for sharing and navigation.

Python HuggingFace (RoBERTa) Streamlit Data Pipelines PDF Export KML Export
Live Application → GitHub Repo → Hugging Face Model → Canonical Data: app/finalData/
Pipeline Architecture
  • • Ingestion: Multi-source travel content is scraped into raw datasets.
  • • Aggregation: Country-level cleaning merges near-duplicates and removes low-information text.
  • • Classification: Fine-tuned RoBERTa model predicts 10 category scores per description.
  • • Finalization: Per-place summarization adds lineage fields and stores canonical outputs for app reads.
  • • UX Layer: Streamlit interface supports filtering, route building, manual reviews, and PDF/KML export.

RoBERTa Aspect Scoring (Distilled)

This model project was created to solve a core travel-planning problem: generic 4.5-star ratings do not explain if a place actually fits a specific traveler profile.

The objective was defined as multi-label aspect scoring from raw review text, producing 10 practical scores (romance, family, adventure, relaxation, price, service, accessibility, nature, culture, food) on a 0-10 scale.

After early synthetic-label attempts with large models proved inconsistent, the final approach used knowledge distillation: a compact RoBERTa backbone (~140M parameters) was trained to reproduce high-quality teacher behavior with lower latency and lower inference cost for production use.

Data + Distillation Workflow
  • • Problem Framing: Recast noisy travel review text into aspect-level scoring instead of star-level ranking.
  • • Data Work: Clean and structure review inputs, enforce consistent category semantics, and standardize targets for model learning.
  • • Failed Path Learned: Pure LLM-generated labels were tested first, then rejected due to unstable quality.
  • • Distillation: Transfer stronger teacher behavior into a smaller RoBERTa student for faster, cheaper inference.
  • • Deployment Fit: Output scores feed directly into RouteMaker personalization and ranking logic.

Smart Home Motion Automation

This project was built to move beyond brittle smart-home "if-this-then-that" rules by analyzing camera activity around the house and generating transparent, auditable automations.

It is a local-first end-to-end engine: YOLOv8-Pose and OpenCV capture motion events, representative moments are selected through cosine-similarity compression, and an Ollama-hosted LLM infers intent before emitting Home-Assistant-ready YAML.

YOLOv8-Pose Ollama (Llama 3.2) OpenCV Few-shot Prompting Cosine Similarity Local-First Privacy YAML Generation
GitHub Repo →
System Pipeline
  • • Capture & Detect: YOLOv8-Pose extracts keypoints and activity cues from camera/video streams.
  • • Compress: Verbose motion logs are condensed into representative events for efficient reasoning.
  • • Intent Prediction: Few-shot LLM stage maps events to user intentions with tunable thresholds.
  • • Action Ranking: Candidate automations are scored and filtered by configurable probability cutoffs.
  • • YAML Emission: Zero-shot generation outputs Home-Assistant-ready automations plus traceable artifacts.

Flight Price Optimizer

Flight Price Optimizer predicts the best booking window for flights using a synthetic but realistic search ecosystem, built because direct scraping of major flight platforms is restricted.

After users choose destination and travel date range, the system generates many randomized flight-search scenarios from those dates (search hour, weekday/weekend behavior, seasonality, holidays, lead time) and runs predictions across all combinations to identify the closest low-price options.

The project includes extensive visualization and explainability: trend charts, distribution plots, correlation views, and SHAP analysis to show which features most influence price forecasts.

Hot Hand Analytics

This project investigates a classic basketball question: does making a shot increase the chance of making the next one, or is the "hot hand" mostly a perception bias?

Using live NBA 2024-25 season data, the analysis focuses on top scorers and compares three-point efficiency after a make versus after a miss through shot-sequence modeling and player-level comparisons.

Results show that most players do not improve meaningfully after a made shot, and some perform better after misses, supporting the hot-hand fallacy as a cognitive illusion rather than a stable statistical effect.

Colab Notebook → GitHub Repo →
Research Design
  • • Data Source: NBA API pull for 2024-25 game and shot context.
  • • Sampling: Filter players by scoring output (PPG) to focus on high-volume shooters.
  • • Core Test: Compare 3PT make probability after make versus after miss.
  • • Visualization: Player-by-player charts expose variance and prevent narrative cherry-picking.
  • • Conclusion: Evidence favors weak or absent hot-hand effect for most players.
